## 🚀 팀원용 '최초' 작업 시작 가이드 (Full Step-by-Step)

이 가이드는 로컬에 프로젝트가 없는 팀원이 **처음 1회** 실행하는 가이드입니다.

### 1단계: 프로젝트 복제 (Clone)

먼저 GitHub에서 프로젝트를 내 컴퓨터로 복제합니다

```bash
git clone https://github.com/SSG-2nd-2team/팀 깃헙 레포지토리.git
```

### 2단계: 폴더 이동 (cd)

방금 복제한 프로젝트 폴더 안으로 이동합니다.

```bash
cd 팀 깃헙 레포지토리
```

### 3단계: 개인 작업 브랜치로 이동 (Switch)

`main` 브랜치가 아닌, Git Master가 미리 만들어 준 **본인의 개인 브랜치**로 이동합니다.

  * (예시: 김도윤님이라면 `dev/KDY`)
  * (예시: 김형근님이라면 `dev/KHG`)

<!-- end list -->

```bash
# 본인 브랜치명으로 바꿔서 입력하세요
예: git switch dev/KHG 
```

> **[\!] 중요:** 이 시점에는 아직 `src`, `build.gradle` 등 **초기 세팅 파일이 보이지 않는 것이 정상**일 수 있습니다. (4단계에서 가져올 것입니다.)

### 4단계: `develop`의 초기 세팅 내용 가져오기 (Merge)

이제 Git Master가 `develop` 브랜치에 올려둔 \*\*'초기 프로젝트 뼈대'\*\*를 내 개인 브랜치로 합쳐서 가져옵니다.

```bash
# 1. GitHub(origin)의 최신 정보를 일단 다 가져옵니다.
git fetch origin

# 2. 'origin/develop' (GitHub의 develop) 내용을
#    '현재 내 브랜치'(dev/KHG)로 병합(merge)합니다.
git merge origin/develop
```

### 💥 5단계: (중요) 4단계 Merge 오류 발생 시 해결

만약 `merge` 명령어 실행 시, 아래와 같은 오류가 발생하면 3단계로 해결하세요.

```
error: The following untracked working tree files would be overwritten by merge:
        .DS_Store
        .idea/misc.xml
Please move or remove them before you merge.
```

이것은 `develop`에 없는 임시 파일이 내 로컬에 있어서 발생하는 문제입니다. 파일을 삭제하고 다시 merge하면 됩니다.

```bash
# 1. (오류가 난 경우) 문제의 임시 파일 삭제
rm .DS_Store
rm .idea/misc.xml 
# (오류 메시지에 나온 파일을 모두 rm 해주세요)

# 2. 다시 merge 명령어 실행
git merge origin/develop

# 3. (Vim 편집기가 뜰 경우)
#    자동으로 입력된 커밋 메시지입니다.
#    키보드에서 :wq (콜론, w, q)를 순서대로 누르고 Enter를 칩니다.
```

### ✅ 6단계: 작업 시작

인텔리제이 재시작 후 폴더 열기

`src/main/java...` 패키지 구조와 `build.gradle` 등 모든 초기 세팅 파일이 보일 겁니다.

이제 본인 브랜치(`dev/KHG`)에서 자유롭게 작업을 시작하시면 됩니다\
